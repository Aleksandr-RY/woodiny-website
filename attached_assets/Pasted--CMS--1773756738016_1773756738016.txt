Нужно переделать админ-панель сайта в полноценную CMS с визуальным управлением контентом.
Цель — чтобы весь лендинг можно было редактировать через админку без изменения кода.

Сейчас проблема:
часть блоков сайта захардкожена в client/landing.html и не управляется админкой.
Из-за этого невозможно нормально менять изображения, карточки и блоки.

Нужно сделать архитектуру как в современных CMS (Webflow / Tilda / Shopify).

---

1. Система блоков страницы (Visual Page Builder)

Создать таблицу blocks.

Поля:

id
type
order
data (JSON)
active

Типы блоков:

hero
clients
features
products
portfolio
process
contacts

Пример data:

{
"title": "Портфолио",
"description": "Наши изделия",
"items": [
{
"title": "Разделочные доски",
"image": "/uploads/board.jpg",
"description": "Из массива дуба"
}
]
}

Лендинг должен загружать блоки через API:

GET /api/blocks

React должен рендерить страницу динамически:

blocks.map(block => renderBlock(block))

Удалить хардкод карточек из client/landing.html.

---

2. Content Collections (карточки контента)

Создать таблицы:

portfolio
products
features
staff

Поля:

id
title
description
image
order
active

Создать страницы админки:

/admin/portfolio
/admin/products
/admin/features
/admin/staff

Функции:

добавление карточек
редактирование
удаление
изменение порядка (drag & drop)

Карточки автоматически отображаются на сайте.

---

3. Media Library (нормальная медиатека)

Использовать существующую загрузку файлов, но сделать полноценную медиатеку:

/admin/media

Функции:

загрузка файлов (drag & drop)
сеточный просмотр изображений
копирование URL
удаление
выбор изображения

При редактировании карточек и блоков должно открываться окно медиатеки,
где можно выбрать изображение.

Изображения должны храниться в:

/uploads/

---

4. Визуальный редактор сайта

Страница:

/admin/editor

Там должен быть список блоков страницы:

Hero
Clients
Features
Products
Portfolio
Process
Contacts

Для каждого блока:

редактирование текста
редактирование изображений
добавление карточек
изменение порядка блоков

Желательно drag & drop для изменения порядка.

---

5. Inline-редактирование текста

В редакторе сайта должно быть удобно менять:

заголовки
подзаголовки
описания

Использовать WYSIWYG редактор (TipTap или аналог).

---

6. API

Добавить API:

GET /api/blocks
POST /api/blocks
PATCH /api/blocks/:id
DELETE /api/blocks/:id

GET /api/portfolio
POST /api/portfolio
PATCH /api/portfolio/:id
DELETE /api/portfolio/:id

GET /api/products
POST /api/products
PATCH /api/products/:id
DELETE /api/products/:id

---

7. Не ломать существующие вещи

Сохранить:

авторизацию
SEO
статистику
заявки
медиатеку
загрузку файлов

---

8. Итог

После изменений админка должна позволять:

менять тексты
менять изображения
добавлять карточки
менять порядок блоков
управлять контентом сайта полностью

без редактирования кода.
