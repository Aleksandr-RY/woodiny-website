После сравнения ZIP версии проекта и GitHub нужно улучшить админ-панель.

Сейчас админка технически работает, но она неудобна для управления сайтом.
Фактически это CRUD интерфейс, а не полноценная CMS.

Цель:
сделать админку, через которую можно управлять всем сайтом без изменения кода.

Нужно реализовать следующие функции.

1. Управление изображениями (Media Library)

Добавить медиатеку.

Функциональность:
- загрузка изображений drag & drop
- предпросмотр
- удаление
- выбор изображения из библиотеки
- копирование URL

Файлы должны храниться в:

/uploads

Backend:
Node.js / Express
использовать multer

Ограничения:

размер файла до 5MB  
форматы:
jpg
jpeg
png
webp

API:

POST /api/upload

Ответ:

{
  "url": "/uploads/filename.jpg"
}

Также нужно добавить:

GET /api/media
DELETE /api/media/:id

2. Редактор контента сайта

Сейчас тексты редактируются неудобно.

Нужно добавить визуальный редактор.

Использовать один из:

TipTap (предпочтительно)
или Editor.js

Редактор должен позволять:

- редактировать текст
- вставлять изображения
- делать списки
- ссылки
- заголовки
- форматирование

Контент должен сохраняться в базе (PostgreSQL).

3. Управление секциями лендинга

Сделать страницу:

"Редактор сайта"

где можно редактировать:

- Hero секцию
- блоки преимуществ
- описание производства
- текстовые блоки
- контакты
- изображения секций

Каждая секция должна храниться в таблице:

siteSettings

Админка должна автоматически подтягивать данные через API.

4. Предпросмотр изменений

Добавить кнопку:

Preview

которая открывает страницу сайта с новыми изменениями без публикации.

5. UX улучшения админки

Добавить:

- загрузку изображений drag & drop
- preview картинок
- индикаторы загрузки
- уведомления об успешном сохранении
- подтверждение удаления

6. Структура

Frontend:
React + shadcn/ui

Backend:
Express + Drizzle ORM

7. Важно

Нельзя ломать текущий деплой.

После изменений код должен:

1) пушиться в GitHub  
2) VPS должен обновляться через

git pull
npm run build
pm2 restart woodiny
